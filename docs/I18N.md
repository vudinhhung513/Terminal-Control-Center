# Đa ngôn ngữ (i18n) — Quy ước BẮT BUỘC

> Tài liệu này mô tả cách hệ thống đa ngôn ngữ hoạt động và **quy ước bắt buộc**
> phải tuân theo khi thêm/sửa giao diện. Mọi text hiển thị cho người dùng phải
> đi qua hệ thống i18n — **không hardcode chuỗi ngôn ngữ trong code/HTML**.

## 1. Nguyên tắc cốt lõi

- **Ngôn ngữ mặc định: tiếng Anh (`en`).** Tiếng Việt (`vi`) là ngôn ngữ thứ hai.
- **Text gốc trong HTML viết bằng tiếng Anh** (làm fallback khi JS chưa chạy).
- Ngôn ngữ lưu **server-side** tại `config.language` (`en` | `vi`), đổi trong
  Settings, áp dụng cho mọi thiết bị.
- Nguồn dịch duy nhất: **`public/js/i18n.js`** (từ điển `DICT`). Không tạo từ
  điển song song ở nơi khác (tránh trùng lặp — xem AGENTS.md mục DRY).

## 2. Thành phần

| Thành phần | Vai trò |
|---|---|
| `public/js/i18n.js` | Từ điển `DICT` + API `window.I18N` (`setLang`, `getLang`, `t`, `apply`). |
| `config.language` | Lưu ngôn ngữ đã chọn (`en`/`vi`), validate trong `config.js`. |
| `GET /api/config` | Trả `language` để client áp dụng khi tải trang. |
| `PUT /api/settings` | Lưu `language` (validate `en`/`vi` trong `routes/settings.js`). |
| Thuộc tính `data-i18n*` | Đánh dấu phần tử HTML cần dịch. |

## 3. API `window.I18N`

```js
window.I18N.setLang('vi');   // dat ngon ngu hien tai
window.I18N.getLang();        // -> 'vi'
window.I18N.t('card.open');   // dich mot key
window.I18N.t('msg.killConfirm', { name: 'web' }); // co bien thay the {name}
window.I18N.apply(root);      // dich moi phan tu data-i18n* trong root (mac dinh document)
```

## 4. Quy ước thêm text mới (BẮT BUỘC)

### 4.1. Text tĩnh trong HTML
Dùng thuộc tính `data-i18n*` thay vì viết chuỗi cứng:

```html
<!-- noi dung -->
<button data-i18n="card.open">Open</button>
<!-- placeholder -->
<input data-i18n-placeholder="toolbar.sessionName" placeholder="Session name">
<!-- title/tooltip -->
<button data-i18n-title="ctrl.enter" title="Enter">⏎</button>
```

Text mặc định trong HTML phải là **tiếng Anh** (khớp giá trị `en` trong từ điển).

### 4.2. Text động trong JS
Dùng `t(key)` (mỗi file frontend đã có shorthand `function t(key, vars)`):

```js
showGlobalMsg(t('msg.createFail') + (err.error || t('msg.unknownError')), 'error');
btn.textContent = t('card.rename');
confirm(t('msg.killConfirm', { name: name })); // bien thay the
```

### 4.3. Thêm key mới
1. Mở `public/js/i18n.js`, thêm vào `DICT`:
   ```js
   'nhom.tenKey': { en: 'English text', vi: 'Văn bản tiếng Việt' },
   ```
2. Đặt key theo **nhóm.camelCase** (vd: `settings.host`, `card.open`, `msg.killFail`).
3. Dùng key qua `data-i18n*` (HTML) hoặc `t()` (JS). **Cả hai ngôn ngữ đều bắt buộc.**

## 5. Luồng áp dụng ngôn ngữ

1. Trang tải → text tiếng Anh hiển thị ngay (fallback từ HTML).
2. JS gọi `GET /api/config`, lấy `language`.
3. `I18N.setLang(language)` rồi `I18N.apply()` → dịch toàn bộ `data-i18n*`.
4. Render động dùng `t()` nên tự đúng ngôn ngữ.
5. Đổi ngôn ngữ trong Settings → lưu qua `PUT /api/settings`, gọi lại
   `setLang` + `apply` + `loadSessions()` → **áp ngay, không reload**.

## 6. Thêm ngôn ngữ thứ ba (vd: `fr`)

1. `config.js`: nới validate `language` để chấp nhận mã mới.
2. `routes/settings.js`: nới validate tương ứng.
3. `public/js/i18n.js`: thêm khoá ngôn ngữ vào **mọi** entry của `DICT`
   (vd thêm `fr: '...'`). Hàm `t()` đã fallback về `en` nếu thiếu khoá.
4. `index.html`: thêm `<option>` vào `#set-language`.
5. Cập nhật tài liệu này + CHANGELOG.
